网页发文后台和发布流程继续完善

2026年4月26日 建站 作者 ZijunQiu-a 约 2 分钟
Contents 目录
网页后台能真正管理文章了草稿和正式发布分开预览更接近公开页面发布状态可以直接看主站也有一些功能增强现在的写作流程

今天继续把 qqqzj@Crane 往“真的能长期写”的方向推进了一轮。前几次主要在打磨首页、阅读体验和静态发布，这一轮重点转到了后台：怎么更方便地写文章、管理文章、预览内容，以及确认网站到底有没有成功发布。

网页后台能真正管理文章了#

现在后台不只是一个新建文章的表单，而是有了文章库。登录以后可以看到所有文章，并按时间从新到旧排列。列表里会显示标题、日期、标签、作者和路径，也能按作者、分类和草稿状态筛选。

这对多人写作很重要。每篇新文章都会写入 author 字段，后台也会根据 GitHub 登录用户判断权限：

  • owner 可以管理所有文章。
  • editor 可以新建文章，也可以修改或删除自己写的文章。
  • editor 不能改别人写的文章。

这样以后如果朋友一起写，也不用担心误删或误改别人的内容。

草稿和正式发布分开#

后台现在把“保存草稿”和“发布文章”拆成了两个按钮。草稿会写入 private source 仓库，但不会出现在公开网站；正式发布才会进入公开构建。

这个变化看起来很小，但写 blog 时会安心很多。文章没写完时可以先存下来，等内容检查完、预览没问题，再正式发出去。

编辑已有文章时，后台会保留原来的文件路径。也就是说，修改标题只会更新 front matter，不会自动改文件名和公开链接。这样可以避免旧链接失效。

预览更接近公开页面#

后台预览继续补齐了 Markdown 支持。现在预览区可以渲染：

  • 标题、段落、列表和代码块
  • 表格、引用和提示块
  • KaTeX 数学公式
  • 图片、视频和附件链接
  • PDF 附件内嵌预览

上传的小文件也会自动进入 posts/media/ 或 posts/files/，并插入对应 Markdown 链接。后端会重新校验文章权限，不允许把文件上传到别人文章对应的资源空间里。

发布状态可以直接看#

后台新增了部署状态面板，会读取 GitHub Actions 里 Deploy Blog 最近一次运行。提交文章后，面板会提示等待部署，再显示最近一次部署成功或失败。

这能减少很多猜测：如果后台显示保存成功，但公开网站还没更新，就可以先看 Actions 状态，而不是一遍遍刷新页面。

这也顺手把 README 更新了一下：后台的 GITHUB_WRITE_TOKEN 如果要读取部署状态，需要有 Actions: Read-only；写文章仍然需要 Contents: Read and write。

主站也有一些功能增强#

这轮不只改后台，主站也补了几个实际使用时会用到的小功能。

搜索结果现在会按相关度排序，并显示匹配片段。比如搜一个关键词时，标题、标签、作者、摘要和正文都会参与匹配，结果里会显示命中的上下文，不再只是普通列表。

文章详情页新增了“回到顶部”和“跳到底部”按钮。长文章读到后面时，可以更快回到标题、目录或上一篇/下一篇区域。

现在的写作流程#

目前比较推荐的流程是：

  1. 打开 Vercel 后台。
  2. 登录 GitHub。
  3. 新建文章，先保存草稿。
  4. 在预览区检查排版、公式、图片和附件。
  5. 正式发布。
  6. 在部署状态里确认 GitHub Actions 成功。

这轮更新之后，网站不再只是一个静态展示页，而是慢慢有了“写作后台 + 自动部署 + 公开阅读”的完整闭环。接下来真正重要的事情，就是继续把内容写进去。
